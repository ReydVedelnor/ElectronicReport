Этап 1. Формализовать бизнес-сценарий

Зафиксируем такие правила
Открытие смены
смена открывается для конкретного подразделения;
смена открывается по конкретному расписанию;
смена открывается конкретным сотрудником;
у подразделения должен быть назначен шаблон отчёта;
при открытии смены автоматически создаётся report_instance.

Заполнение отчёта
фронт получает шаблон не как “голые таблицы”, а как готовую структуру формы;
для каждого атрибута фронт видит:
id
название
тип узла (section / metric)
required
data type
unit
sort order
numbering
display style
текущее значение

Завершение смены
смену нельзя завершить, если обязательные поля не заполнены;
при завершении:
отчёт получает статус ready
смена получает статус closed
у смены выставляется ended_at
у отчёта выставляется closed_at

Передача смены
передать можно только закрытую смену;
передача создаётся между from_shift_id и to_shift_id;
следующая смена должна существовать;
передача может быть sent, accepted, rejected.
