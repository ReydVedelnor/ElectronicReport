Создай актуальный JSON-контекст проекта для переноса в новый чат с GPT/LLM.

Цель: после загрузки JSON новая модель должна понять текущее состояние проекта и продолжить разработку без восстановления контекста.

На входе:
- устаревшее описание проекта;
- актуальная файловая структура;
- текущие договорённости из диалога;
- при необходимости — фрагменты кода.

Сформируй валидный JSON на русском языке.

Включи только:
- стек и архитектурные правила;
- структуру модулей на высоком уровне;
- ключевые entity/repository/use-case/DTO сведения;
- текущие бизнес-правила;
- актуальные API-контракты;
- принятые решения пользователя;
- текущий статус реализации;
- что подтверждено, что запланировано, что нужно проверить по коду;
- риски компиляции/бизнес-логики;
- ближайшие следующие шаги.

Не включай:
- историю обсуждения;
- рассуждения;
- повторы;
- полную файловую структуру, если она есть отдельным файлом;
- примеры response, если они не критичны;
- устаревшие решения, кроме явно помеченных как deprecated;
- неподтверждённые реализации без пометки `verify_in_code`.

Если есть неопределённость — не выдумывай, а добавь в `unknown_or_needs_verification`.

Верни только JSON без markdown.
